---
title: "Identity & key status"
description: "Check who a key belongs to, whether it's valid, and when it expires."
---

A few lightweight endpoints let you inspect the key you're calling with — useful for health
checks, diagnostics, and surfacing key state in your own UI.

## Who am I?

`POST /public/whoAmI` returns the identity behind the current key — the quickest way to
confirm a key works and see which account it maps to.

```bash
curl https://dev.api.nestapi.com/public/whoAmI \
  -X POST \
  -H "Authorization: Bearer YOUR_API_KEY"
```

## Is the key valid?

`GET /auth/validate` confirms the key is active and returns basic identity info
(`Valid`, `UserId`).

```bash
curl https://dev.api.nestapi.com/auth/validate \
  -H "Authorization: Bearer YOUR_API_KEY"
```

## When does the key expire?

`GET /apiKey/validUntil` returns the key's expiry. A key with no expiry returns
`31/12/9999`.

```bash
curl https://dev.api.nestapi.com/apiKey/validUntil \
  -H "Authorization: Bearer YOUR_API_KEY"
```

<Tip>
Poll `validUntil` in scheduled jobs to warn before an integration's key lapses, and rotate
ahead of time with `PUT /apiAdmin/keys` (see [API keys](/account/api-keys)).
</Tip>

<Note>
Managing keys themselves (create, list, rotate, delete) needs an Admin-class key — see
[API keys](/account/api-keys).
</Note>
